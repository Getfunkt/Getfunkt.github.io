Mineral Recon

Mineral Recon is a gamejam project i made with my brother for Ryan Laley Games Summer Jam 2023 with a theme of contrast and a timelimit of 48 hours.

Link to the gamejam project Mineral Recon.

Mineral Recon was made with Unreal Engine 5.3 and models were made with Blender 3.2.

In the game player pilots a drone with a drill and a weapon with the objective to mine as many minerals they can while also avoiding or shooting threats on an asteroid. The drone moves on the asteroid like a spider clinging to the surface. The theme contrast is used as a gameplay mechanic that works by making the player adjust the contrast with the mouse wheel to keep the visibility clear while moving on the asteroid.

What i did in the project:

Visuals and the logic for the User Interface. Below is an image showcasing the UI

Weapon and drill functionality for the player

3D models and animations made with blender.

Magic system

The magic casting system made allows for easier customisation of casting properties.

The system is a framework for creating new magic logic and creating the appropriate interactions between characters and the magic. It separates casting from the magic functionality and adds a layer of customisation.

Its made in C++ with blueprint integration.

The Blueprint of the projectile magic only requires the logic for spawning it and giving the settings from the data asset.

Projectile magic is set to fire rapidly at 1000 speed every 0.05 seconds and will stop firing after releasing.

Or it can be set to fire one when mouse button is released with lower speed and high gravity.

A speed modifier can be configured to add 800 speed after the two second cast time is done and it is reactivated and will end after another reactivation.

An area magic that is set up to start casting the magic as soon as it is activated and it cannot be selected. It will then spawn the magic area as soon as the casting time ends. The magic area is destroyed when the magic is reactivated.

A wind magic is set to be selectable and the direction can be adjusted by holding the mouse button down.

VR Store Game

VR Store Game is an game made by me for an school course where the idea was to make an educational game in VR environment.

.

The project was made with Unreal Engine 5.4 blueprints and developed with both HTC Vive- and Valve Index headsets

Project overview:

VR Store is a VR game about managing a store. The main tasks include restocking shelves, guiding customers to products, scanning, and handling payment at the cash register.

Main Features:

Dynamic shelving for the products. It works by taking the bounds of the mesh and making a grid from it.

Customer AI walks around the store picking up products and when finished it goes to the cash register to pay for the products. Sometimes AI can't find a product, and player must go help guide the customer AI to a product.

Products are one blueprint that is populated by a data table where you can easily add more products.

Cash register blueprint where you can scan products, and they are added to an in-world UI.

I learned from this project to develop VR projects using Unreal Engine and developing modular systems such as product blueprint and dynamic shelves.
